# Security Policy

## Supported Versions

当前优先维护 `main` 分支最新版本。历史版本默认不再提供安全更新。

## Reporting a Vulnerability

请不要在公开 Issue 中直接披露漏洞细节。

建议流程：

1. 使用 GitHub 仓库的 Security -> Advisories 提交私密报告。
2. 或创建带有最小复现信息的私密沟通（包含影响范围、复现步骤、建议修复方式）。

我们会在收到报告后尽快确认并给出处理状态：

- 48 小时内：确认收到
- 7 天内：完成初步评估与修复计划
- 修复发布后：在变更日志中注明安全修复条目（必要时附 CVE）

## Scope

安全问题包括但不限于：

- 未授权访问
- 密码或令牌处理缺陷
- 敏感信息泄露
- 任意路径写入/读取风险
- 依赖供应链高危漏洞
